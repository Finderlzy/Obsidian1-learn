## 理解

类就是创建一种数据类型，和[[Sequence Container|vector]]、[[Container Adaptor|Stack]]、[[Container Adaptor|Queue]]一样。

## 设计类

在设计一个类的时候，一般要考虑三个东西：**成员变量**、、**成员函数（方法）**、**构造**。

### 成员变量

就是你这个类要用来存放、处理哪些数据，这些数据又是哪些类型。

eg.如果我要创建一个银行卡账户的类，那么我的成员变量会是：卡号、余额。

### 成员函数

就是你这个类可以进行哪些操作。

eg.还是银行卡账户的类，可以进行的操作有：余额增加（add），转账（transfer）。
还可以有别的，只是我暂时想到这些

### 构造

就是如何创建你自己定义的这个类。

默认的方法就是创建普通变量的方法，但是这样创建的话成员变量都是空，所以有时会需要构造。

## 写类

写一个类要完成2件事：**写头文件（.h）**、**写实现文件（.cpp)**。

### 写头文件
```
BankAccount.h

#include <iostream.h>
#program once

class BankAccount{

// public一般放方法，用于操作类
public:
void add(int money);
void transfer(string bankId, int money);


// private一般放变量，防止被用户操作
private:
int balance; -->余额
string id;   -->卡号

};
```

这就是写头文件，就是告诉编译器我将要写哪些东西

### 写实现文件

就是具体定义和实现那些方法。

需要注意的是，定义函数要像下面那样写：
```
void BankAccount::add(int money){
  balance += money;
}
```

就是==要多写“BankAccount ::”来告诉编译器，我要写的是BankAccount这个类的add函数==

## 使用类

使用类的时候，我们只能用public里的函数，也叫做“接口”来操作。我们并不关心接口是如何实现的，用就行了。

但同时，我们也无法操作类的数据。

==这就是[[抽象]]的思想==
